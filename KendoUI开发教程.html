1.1 单页面应用（一） -- 概述
Router类负责跟踪应用的当前状态和在不同的应用状态之间切换。Router类通过url的片段功能和流量器的浏览历史功能，可以实现把应用的某个状态作为书签添加到浏览器中。
View类和Layout类负责UI的显示。
UI事件或数据绑定可以通过MVVM或data初始化属性来完成

<div id="app"></div>  //定义View挂载点

<script id="index" type="text/x-kendo-template"> //定义View使用的模板
     Hello <span data-bind="text: foo"></span>
</script>

<script>
     //创建View，用于显示UI组件的显示
     //一个View包含两个部分：这个View使用的模板(模板ID)、这个模板使用的数据源
     var index = new kendo.View( "index", 
          {
               model: kendo.observable({
                    foo: "World!"
               })
          }
     );
     //创建Router类，用于导航
     var router = new kendo.Router();

     //定义路径 和 这个路径使用的View（在这个View中设置挂载在哪个HTML元素下） 
     router.route("/", function() {
          index.render("#app");
     });
 
     $(function() {
          //启动导航 
          router.start();
     });
</script>


1.2 单页面应用（二） -- Router类
Router类支持通过代码在应用的不同状态之间切换
<script>
     var router = new kendo.Router();
     router.route("/", function() {
          console.log("/ url was loaded");
     });
     $(function() {
          router.start();
          //router.navigate("/xxxx"); 
     });
</script>
缺省状态下（没有调用router.navigate()函数），使用根路径（'/'），此时根路径对应的回调函数被调用。

参数：绑定参数、可选参数、匹配符匹配参数
绑定参数
<script>
     var router = new kendo.Router();
     router.route("/items/:category/:id", function(category, id) {
          console.log(category, "item with", id, " was requested");
     });
     $(function() {
          router.start(); 
          router.navigate("/items/books/59");
     });
</script>
category被绑定为books、id被绑定为59

可选参数
<script>
     var router = new kendo.Router();
     router.route("/items(/:category)(/:id)", function(category, id) {
          console.log(category, "item with", id, " was requested");
     });
     $(function() {
          router.start(); // ...
          router.navigate("/items/books/59"); // ...
          router.navigate("/items"); // ...
          router.navigate("/items/books");
     });
</script>

匹配符匹配参数
<script>
     var router = new kendo.Router();
     router.route("/items/*suffix", function(suffix) {
          console.log(suffix);
     });
     $(function() {
          router.start(); // ...
          router.navigate("/items/books/59"); // ...
          router.navigate("/items/accessories"); // ...
          router.navigate("/items/books");
     });
</script>

页面切换
navigate()方法可以用来切换页面，对应路径的回调方法被调用。调用navigate()方法会修改URL fragment（#后面的部分）
<script>
     var router = new kendo.Router();
     router.route("/foo", function() {
          console.log("welcome to foo");
     });
     $(function() {
          router.start();
          router.navigate("/foo");
     });
</script>

可以使用链接，触发router.route()
<a href="#/foo">Foo</a>

如果对应的路径不存在，则触发Router类的routerMissing事件，在回调函数的参数中封装了url
<script>
     var router = new kendo.Router({
          routeMissing: function(e) {
               console.log(e.url)
          }
     });
     $(function() {
          router.start();
          router.navigate("/foo");
     });
</script>

可以通过Router类的change事件，来截获页面之间的切换。也可以通过阻止组件的默认行为，来阻止页面的切换
<script>
     var router = new kendo.Router({
          change: function(e) {
               console.log(e.url);
               e.preventDefault();
          }
     });
     $(function() {
          router.start();
          router.navigate("/foo");
     });
</script>





2.1 单页面应用（三） -- View
View类负责UI组件的显示，View的显示可以通过HTML创建，也可以通过script模板创建

使用html创建View
<script>
     var index = new kendo.View('<span>Hello World!</span>');
</script>

使用script创建View 
<script id="index" type="text/x-kendo-template">
     <span>Hello World!</span>
</script>
<script>
     var index = new kendo.View('index');
</script>

向某个HTML元素挂在View内容 -- render()
<div id="app"></div>
<script>
     var index = new kendo.View('<span>Hello World!</span>');
     index.render("#app");
</script>

向某个HTML元素中添加View的内容 --append()
<div id="app"></div>
<script>
     var index = new kendo.View('<span>Hello World!</span>');  
     $("#app").append(index.render());
</script>


集成MVVM
<div id="app"></div>
<script id="index" type="text/x-kendo-template">
     <div>Hello <span data-bind="text:foo"></span>!</div>
</script>
<script>
     var model = kendo.observable({ foo: "World" });
     var index = new kendo.View('index', { model: model });
     index.render("#app");
</script>



3.1 单页面应用（四） -- Layout
Layout集成自View，可以用来包含其它View或Layout

<div id="app"></div>
<script>
     var bo = new kendo.View("<span>Body</span>");
     var fo = new kendo.Layout("<span>Footer</span> ");
     var layout = new kendo.Layout("<header>Header</header><section id='bo'></section><footer id='fo'></footer>");
     layout.render($("#app"));
     layout.showIn("#bo",bo);
     layout.showIn("#fo",fo);
</script>

数据绑定  data-bind

创建时间：2016-9-12 15:57
更新时间：2016-9-13 10:09
作者：李洪岩

kendoUI MVVM的数据绑定支持的属性有：
     attr 、checked 、click 、custom 、events 、text 、value 、
     style 、html 、enable 、disable 、visibile 、invisible 

1.attr  用于绑定HTML元素的属性
attr支持把ViewModel的属性或方法绑定到HTML元素的某个属性
基本用法：attr:{attribute1:field1,attribute2:field2}，其中attribute1和attribute2为HTML元素的属性名称，field1和field2为ViewModel对象的属性名称。
<img id="logo" data-bind="attr:{ src: imageSource,alt: imageAlt }" />
<script>
     var viewModel = kendo.observable({
          imageSource: "http://www.kendoui.com/image/kendo-logo.png", imageAlt: "Kendo Logo" });          
          kendo.bind($("#logo"), viewModel);
</script>
与<img id="logo" src:="imageSource" alt="imageAlt" />等价


attr支持绑定HTML的数据属性
<div data-bind="attr: { data-foo: fooValue, data-bar: barValue }"></div>
<script>
     var viewModel = kendo.observable({ fooValue: "foo", barValue: "bar" });
     kendo.bind($("div"), viewModel);
</script>
与<div data-foo="fooValue" data-bar="barValue"></div>等价


注意：如果需要绑定HTML元素的value属性或者checked属性，需要使用kendo MVVM 的 value 和 checked 绑定方法



2.1 checked  用于绑定checkbox、radio是否被选中
注：
//单选按钮 -- 一个按钮、选或不选 （方形）
<input type="checkbox">  

//单选按钮 -- 多个按钮、选择一个 （圆形）
<input type="radio" value="Red" name="color" />Red
<input type="radio" value="Green" name="color" />Green
<input type="radio" value="Blue" name="color" />Blue

//多选按钮 -- 多个按钮、选多个 (方形)
<input type="checkbox" value="Red" name="color" />Red
<input type="checkbox " value="Green" name="color" />Green
<input type="checkbox " value="Blue" name="color" />Blue

checked绑定只适用于支持checked的HTML元素，其他HTML元素的绑定可以使用value绑定


2.2 绑定一个数组到一组多选框 -- checkbox
<input type="checkbox" value="Red" data-bind="checked: colors" />Red
<input type="checkbox" value="Green" data-bind="checked: colors" />Green
<input type="checkbox" value="Blue" data-bind="checked: colors" />Blue
<script>
     var viewModel = kendo.observable({
          colors: ["Red"]
     });
     kendo.bind($("input"), viewModel);
</script>
因为第一个checkbox的value包含在ViewModel，第一个checkbox为选中状态。当其他checkbox为选中状态时，对应的checkbox的value被加入到ViewModel的数组中


2.3 绑定一个值到单选框 -- radio
<input type="radio" value="Red" name="color" data-bind="checked: selectedColor" />Red
<input type="radio" value="Green" name="color" data-bind="checked: selectedColor" />Green
<input type="radio" value="Blue" name="color" data-bind="checked: selectedColor" />Blue
<script>
     var viewModel = kendo.observable({
          selectedColor: "Green"
     });
     kendo.bind($("input"), viewModel);
</script>



3.1 使用click绑定
click绑定是event的一种特例，用于将HTML的点击事件绑定到ViewModel的函数
形式一：
     <span data-bind="click: showDescription">Show description</span>
形式二：
     <span data-bind="events: { click: clickHandler }">Show description </span>


3.2 终止事件向上传递
使用e.stopPropagation()函数

<span data-bind="click: click">Click</span>
<script>
     var viewModel = kendo.observable({
          click: function(e) {
               e.stopPropagation();
          }
     });
     kendo.bind($("span"), viewModel);
</script>


3.3 停止HTML中某些元素对单击事件的缺省处理
如果要取消某些HTML元素对单机事件的缺省处理，例如跳转到其他页面或是提交表单，可以使用preventDefault()函数

<a href="http://example.com" data-bind="click: click">Click</span>
<script>
     var viewModel = kendo.observable({
          click: function(e) {
               // stop the browser from navigating to http://example.com
               e.preventDefault();
          }
     });
     kendo.bind($("a"), viewModel);
</script>




4.1 enabled/disabled
enable/disable仅适用于input、select、textarea。当这些元素disable=true时，无法修改其值

<div id="view">
     <input type="text" data-bind="value: name, disabled: isNameDisabled" />
     <button data-bind="click: disableInput">Disable</button>
</div>
<script>
     var viewModel = kendo.observable({
          isNameDisabled: false,
          name: "John Doe",
          disableInput: function () {
               this.set("isNameDisabled", true);
          }
     });
     kendo.bind($("#view"), viewModel);
</script>


注意：0、null、undefined均被当成false，其他值当成true

8.1 style绑定--为HTML元素追加样式
<span data-bind="style: {color: priceColor, fontWeight: priceFontWeight}, text: price"></span>
<script>
     var viewModel = kendo.observable({
          price: 42,
          priceColor: function() {
               var price = this.get("price");
               if (price <= 42) {
                    return "#00ff00";
               } else {
                    return "#ff0000";
               }
          },
          priceFontWeight: function() {
               var price = this.get("price");
               if (price <= 42) {
                    return "bold";
               } else {
                    return ""; //will reset the font weight to its default value
               }
          }
     });
     kendo.bind($("span"), viewModel);
</script>

注意：如果CSS属性中含有连字符（-），例如font-weight、font-size，在绑定时，需要省略连字符，使用camel风格的命名方式：fontWeight、fontSize。



9.1 text - 为HTML元素追加文本属性
注意：text绑定可以为HTML元素设置文本属性，如果要为input、select、textarea设置文本属性，需要使用value绑定。如果ViewModel的属性中包含HTML标签，使用text会直接显示出来。如果想编译HTML标签，可以使用html绑定。

<span data-bind="text: name"></span>
<script>
     var viewModel = kendo.observable({
          name: "John Doe"
     });
     kendo.bind($("span"), viewModel);
</script>



10.1 source - 与kendoUI模板结合
source绑定ViewModel的数组属性
source绑定可以把ViewModel的值与kendoUI模板定义的目标元素绑定；模板由属性data-template指定，它的值为某个script模板的id，如果没有指定模板，则根据元素的标记使用缺省模板。

<ul data-template="ul-template" data-bind="source: products"></ul>
<script id="ul-template" type="text/x-kendo-template">
     <li>
          id: <span data-bind="text: id"></span>
          name: <span data-bind="text: name"></span>
     </li>
</script>
<script>
     var viewModel = kendo.observable({
          products: [
               { id: 1, name: "Coffee" },
               { id: 2, name: "Tea" },
               { id: 3, name: "Juice" }
          ]
     });
     kendo.bind($("ul"), viewModel);
</script>

如果ViewModel绑定的数组为简单类型，这时在模板中可以使用"this"关键字来引用当前数据项
<ul data-template="ul-template" data-bind="source: products"></ul>
<script id="ul-template" type="text/x-kendo-template">
     <li data-bind="text: this"></li>
</script>
<script>
     var viewModel = kendo.observable({
          products: [ "Coffee", "Tea", "Juice" ]
     }); kendo.bind($("ul"), viewModel);
</script>

source绑定ViewModel的非数组属性
source绑定也支持绑定非数组属性，最后的结果为模板使用ViewModel的结果
<script id="div-template" type="text/x-kendo-template">
     Name: <span data-bind="text: name"></span>
</script>
<div data-template="div-template" data-bind="source: person"></div>
<script>

          person: { name: "John Doe" }
     });
     kendo.bind($("div"), viewModel);
</script>

source绑定也可以直接绑定到ViewModel对象本身，此时可以把source的值设置为this
<script id="div-template" type="text/x-kendo-template">
     Name: <span data-bind="text: person.name"></span>
</script>
<div data-template="div-template" data-bind="source: this"></div>
<script>
     var viewModel = kendo.observable({
          name: "John Doe"
     });
     kendo.bind($("div"), viewModel);
</script>

11.1 value绑定
value绑定用于把ViewModel的属性绑定到HTML元素的value中，因此value绑定只适用于支持value属性的元素：input、select、textarea；支持value属性的UI组件：AutoComplete、DropDownList等。
如果需要设置HTML元素的文本或html内容，需要使用text和html绑定；对于checkbox、radio需要使用checked绑定。

11.2 input和textarea的value绑定
value绑定用于把ViewModel的属性绑定到HTML元素的value中，因此value绑定只适用于支持value属性的元素：input、select、textarea；支持value属性的UI组件：AutoComplete、DropDownList等。
<div id="view">
     <input data-bind="value: inputValue" />
     <textarea data-bind="value: textareaValue"></textarea>
</div>
<script>
     var viewModel = kendo.observable({
          inputValue: "Input value",
          textareaValue: "Textarea value"
     });
     kendo.bind($("#view"), viewModel);
</script>
value绑定默认依赖于HTML元素的change事件，也就是HTML元素失去焦点。当HTMl失去焦点时，input、textarea、select的改变反映到ViewModel中，反之亦然。可以使用data-value-update属性来使用不同的事件。
<div id="view">
     <input data-value-update="keyup" data-bind="value: inputValue" />
</div>
<script>
     var viewModel = kendo.observable({
          inputValue: "Input value"
     });
     kendo.bind($("#view"), viewModel);
</script>





11.3 value绑定select
当select元素使用了预定义的选项时，MVVM将根据ViewModel的值将和该值相同的option设定为选中状态。当更改option时，ViewModel的对应属性也会更改。反之亦然。
<select data-bind="value: selectedColor">
     <option value="red">Red</option>
     <option value="green">Green</option>
     <option value="blue">Blue</option>
</select>
<script>
     var viewModel = kendo.observable({
          selectedColor: "green"
     });
     kendo.bind($("select"), viewModel);
</script>

如果option没有定义value，那么使用option的text属性
<select data-bind="value: selectedColor">
     <option>Red</option>
     <option>Green</option>
     <option>Blue</option>
</select>
<script>
     var viewModel = kendo.observable({
          selectedColor: "green"
     });
     kendo.bind($("select"), viewModel);
</script>


如果select支持多项选择，那么ViewModel的属性为一个数组，并且select添加multiple="multiple"属性

1.ObservableObject对象
MVVM中的一个关键部分是ViewModel，ViewModel主要是通过kendo.data.ObservableObject来提供支持的，ObservableObject可以监控改变并通知关心该变化的组件。为了创建一个ObservableObject对象，可以使用kend.Observable或者kendo.data.ObservableObject，当使用kendo.Observable创建ViewModel对象，在kendo.bind方法中，kendoUI会将传入的ViewModel对象转化为ObservableObject对象，除非传入的就是一个ObservableObject对象。

如果创建的ViewModel对象在后面还会用到，则必须使用kendo.Observable或者kendo.data.ObservableObject。
如果创建的ViewModel对象仅使用一次，可以使用JavaScript对象创建

1.准备开始
使用MVVM模式，首先创建ViewModel对象，ViewModel代表可以使用View显示的数据对象。在kendoUI中，使用kendo.observable函数通过传入JavaScript对象的方法来定义一个ViewModel对象
var viewModel = kendo.observable({
     name:"John Doe",
     displayGreeting:function(){
          varname = this.get("name");
          alert("Hello, "+ name +"!!!");
     }
});

使用HTML创建一个View，在这个View中包含一个文本框和一个按钮。文本框通过data-bind属性绑定到observable的name，name值得变化将会反映到文本框中，反之亦然。按钮的点击事件绑定到displayGreeting函数。
<div id="view">
     <input data-bind="value: name" />
     <button data-bind="click: displayGreeting">Display Greeting</button>
</div>

最后，通过kendo.bind方法将View和ViewModel绑定起来
kendo.bind($("#view"), viewModel)
----------------------------------------------------

<div id="view">
     <input data-bind="value: name" />
     <button data-bind="click: displayGreeting">Display Greeting</button>
</div>
<script>

          name: "John Doe",
          displayGreeting: function () {
               var name = this.get("name");

          }
     });

2.数据绑定
数据绑定将HTML元素绑定到ViewModel对象的某个属性或方法，绑定通过data-bind属性实现，采用"绑定类型:ViewModel的属性和方法"的格式。kendoUI支持的绑定类型有：source、attr、html、text、style、value、disable/enable、visible/invisible等。data-bind也支持使用逗号分隔的属性列表。kendoUI MVVM支持嵌套的ViewModel属性
例如：将div绑定到person.name
<div data-bind="text: person.name"></div>
<script>

          }

应该这样使用

     });
     kendo.bind($("div"), viewModel);

1.将一个HTML元素转化为一个KendoUI控制项
1.使用JQuery插件的方法初始化
  <input id="datepicker"/>
  $("#datepicker").kendoDatePicker();
2.通过data属性初始化
  需要设置data的role属性，然后调用kendo.init()方法
<div id="container">
     <input data-role="numerictextbox" />
</div>
<script>
     kendo.init($("#container"));
</script>

2.配置
每个组件都具有一些属性，可以通过data属性对该组件进行配置，使其具有某种行为，配置原则：
1.对于组件的简的属性，直接在该属性前添加"data-"前缀，例如delay属性->data-delay。
2.对于组件中以驼峰式命名的属性，通过添加"-"分割，例如AutoComplete的ignoreCase属性可以通过data-ignore-case设置。
3.对于属性是以data开头的属性，不用添加"data-"前缀，直接以"-"分割，例如：dataTextField属性->data-text-field；dataSource属性->data-source


<div id="container">
     <input data-role="autocomplete" data-ignore-case="true" data-text-field="name" data-source="{data: [{name: 'John Doe'},{name: 'Jane Doe'}]}" />
</div>

<script>
     kendo.init($("#container"));
</script>


3.事件
可以通过data属性添加对kendoUI组件的事件处理，属性值是一个JavaScript函数，作用域为全局；事件处理函数也可以是一个对象的成员函数。

data-change事件以作用域失去焦点衡量

<div id="container">
     <input data-role="numerictextbox" data-change="numerictextbox_change" />
</div>
<script>
     function numerictextbox_change(e) {
          // Handle the "change" event
     }
     kendo.init($("#container"));
</script>
-------------------------------------------------------
<div id="container">
     <input data-role="numerictextbox" data-change="handler.numerictextbox_change" />
</div>
<script>
     var handler = {
          numerictextbox_change: function (e) {
               // Handle the "change" event
          }
     };
     kendo.init($("#container"));
</script>


4.数据源
支持数据绑定的kendoUI组件的数据源可以通过data-source属性进行指定，data-source属性的值为JavaScript对象、数组、全局变量
1.使用JavaScript数组作为数据源
<div id="container">
    <input data-role="autocomplete" data-source="{data:['One', 'Two']}" />
</div>
<script>
     kendo.init($("#container"));
</script>

<div id="container">
    <input data-role="autocomplete" data-source="['One', 'Two']" />
</div>
<script>
     kendo.init($("#container"));
</script>

2.使用一个可以全局访问的变量作为数据源
<div id="container">
    <input data-role="autocomplete" data-source="dataSource" />
</div>
<script>
     var dataSource = new kendo.data.DataSource( {
         data: [ "One", "Two" ]
     });
     kendo.init($("#container"));
</script>



5.模板
可以使用data元素来定义模板，属性值为用来定义模板的script的id（需要为script添加type="text/x-kendo-template"属性）

<div id="container">
    <input data-role="autocomplete"
           data-source="[{firstName:'John', lastName: 'Doe'}, {firstName:'Jane', lastName: 'Doe'}]"
           data-text-field="firstName"
           data-template="template" />
</div>
<script id="template" type="text/x-kendo-template">
    <span>#: firstName # #: lastName #</span>
</script>
<script>
     kendo.init($("#container"));
</script>
